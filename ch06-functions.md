## 6.1 函数：代码的“积木”

函数是执行特定任务的代码块，可以接收输入并返回输出。它让代码复用成为可能。

**求最大公约数（欧几里得算法）**

```cpp
int gcd(int m, int n) {
    while (n != 0) {
        int r = m % n;
        m = n;
        n = r;
    }
    return m;
}

int main() {
    std::println("{}", gcd(72, 27));  // 输出 9
}
```

**函数声明与定义**：声明告诉编译器函数存在（常放在头文件），定义给出具体实现。大型项目中两者必须分开，遵循**单一定义规则（ODR）**。

> 完整版会详细讲解多文件项目结构、头文件保护、以及`#include`的正确用法。

## 6.2 参数传递：值传递 vs 引用传递

- **值传递**：形参是实参的副本，修改形参不影响实参。
- **引用传递**：形参是实参的别名，修改形参直接改变实参。

```cpp
void swap(int& a, int& b) {   // 引用传递
    int t = a;
    a = b;
    b = t;
}

int x = 1, y = 2;
swap(x, y);   // x=2, y=1
```

**数组形参**：实际上是指针，不能直接使用范围`for`。需要额外传递大小。

```cpp
void print(int arr[], int n) {  // 等价于 int* arr
    for (int i = 0; i < n; ++i)
        std::print("{} ", arr[i]);
}
```

**`const`与形参**：用`const`修饰形参可防止意外修改，提高代码安全性。

```cpp
void readOnly(const std::string& s) { /* 不能修改 s */ }
```

> 完整版会讲解指针传参、`const`的四种组合（指向常量的指针、常量指针等）、以及默认参数的规则。

## 6.3 返回值：`return`的学问

函数可以返回值、引用或指针。

**返回引用**：可以修改被引用的对象，但**绝不能返回局部变量的引用**（悬垂引用）。

```cpp
int& getElement(int arr[], int index) {
    return arr[index];   // 安全：arr的生命周期由调用者管理
}
```

**返回值优化（RVO）**：现代编译器会避免不必要的拷贝，直接在调用者的内存中构造返回值。

```cpp
std::string makeString() {
    return "hello";   // 直接构造在外部，无拷贝
}
```

> 完整版会深入讲解返回指针、`const`返回值、以及如何安全地返回动态分配的内存。

## 6.4 静态变量：跨调用保持状态

**静态局部变量**：生命周期贯穿程序运行，但作用域仅限于函数内部。常用于计数器、缓存。

```cpp
int counter() {
    static int count = 0;   // 只初始化一次
    return ++count;
}
// 连续调用输出：1, 2, 3, ...
```

**文件作用域的静态变量**：限制变量仅在当前文件可见，避免命名冲突。

> 完整版会对比静态局部变量与全局变量，并展示其在记忆化递归（斐波那契）中的应用。

## 6.5 函数重载：同名不同参数

允许定义多个同名函数，只要参数列表不同（个数、类型、顺序）。**返回类型不能作为重载依据**。

```cpp
void print(int x) { std::println("int: {}", x); }
void print(double x) { std::println("double: {}", x); }
void print(const std::string& s) { std::println("string: {}", s); }
```

**重载解析**：编译器根据实参选择最佳匹配（精确匹配 → 类型提升 → 标准转换 → 自定义转换）。

> 完整版会讲解`const`对象的引用/指针在重载中的特殊规则，以及常见陷阱（如`int`与`long`的歧义）。

## 6.6 内联函数 vs 宏

**内联函数**（`inline`）：建议编译器将函数调用展开为函数体，消除调用开销。适用于短小、频繁调用的函数。

```cpp
inline int square(int x) { return x * x; }
```

**宏**（`#define`）是文本替换，无类型检查，极易出错，**现代C++中应避免使用**。

```cpp
#define SQUARE(x) (x * x)   // 危险：SQUARE(1+1) 展开为 1+1*1+1 = 3
```

> 完整版会对比内联函数与宏的6大区别，并说明编译器对内联请求的自主权。

## 6.7 `constexpr`：编译期计算

`constexpr`变量/函数的值在编译期确定，可用于数组大小、模板参数等。

```cpp
constexpr int factorial(int n) {
    int result = 1;
    for (int i = 2; i <= n; ++i) result *= i;
    return result;
}
int arr[factorial(5)];   // 编译期确定大小：120
```

C++20进一步引入`consteval`（强制编译期执行）和`constinit`（编译期初始化，运行时可修改）。

> 完整版会对比`const`、`constexpr`、`consteval`、`constinit`的适用场景，并给出编译期字符串处理的例子。

## 6.8 递归：函数调用自身

递归将大问题分解为同类的子问题，必须包含**基情形**（终止条件）和**递归情形**。

**汉诺塔**：经典递归问题。

```cpp
void hanoi(int n, char from, char to, char aux) {
    if (n == 1) {
        std::println("Move disk 1 from {} to {}", from, to);
        return;
    }
    hanoi(n - 1, from, aux, to);
    std::println("Move disk {} from {} to {}", n, from, to);
    hanoi(n - 1, aux, to, from);
}
```

**注意**：递归深度过大可能导致栈溢出，且存在重复计算（如斐波那契），可用记忆化优化。

> 完整版包含斐波那契记忆化、快速排序递归实现、迷宫求解（回溯法）等完整代码。

## 6.9 函数指针与回调

函数指针可以像变量一样传递，实现回调机制。

```cpp
int add(int a, int b) { return a + b; }
int sub(int a, int b) { return a - b; }

int compute(int (*op)(int, int), int x, int y) {
    return op(x, y);
}

int main() {
    std::println("{}", compute(add, 5, 3));   // 8
    std::println("{}", compute(sub, 5, 3));   // 2
}
```

**函数指针数组**：可动态选择调用的函数，避免冗长的`if-else`。

> 完整版会讲解函数指针的类型别名（`using`）、与`std::function`的区别，以及无捕获Lambda到函数指针的隐式转换。

## 6.10 Lambda 表达式：匿名函数

C++11引入的Lambda可以在调用点直接定义函数对象，特别适合STL算法。

```cpp
std::vector<int> v{3, 1, 4, 1, 5};
std::sort(v.begin(), v.end(), [](int a, int b) {
    return a > b;   // 降序
});
```

**捕获列表**：`[=]`按值捕获，`[&]`按引用捕获，`[x, &y]`混合捕获。

**`mutable`**：允许修改按值捕获的副本。

**初始化捕获（C++14）**：在捕获列表中定义并初始化新变量。

```cpp
auto counter = [count = 0]() mutable { return ++count; };
std::println("{}", counter());  // 1
std::println("{}", counter());  // 2
```

> 完整版会详细讲解Lambda的语法、无捕获Lambda可转换为函数指针、以及在`std::thread`、`std::async`中的应用。

## 6.11 实战：字符图形库 ChGL 与 Pong 游戏

**ChGL**是一个基于字符的图形库，使用帧缓冲区（一维字符数组）模拟像素，通过`setPixel`绘制点，`show`一次性输出。

**核心优化**：每行末尾预存换行符，一次`fwrite`完成所有输出，实现无闪烁动画。

**Pong游戏**：基于ChGL实现双人乒乓球游戏，支持跨平台键盘输入（W/S控制左挡板，方向键控制右挡板，ESC退出），包含碰撞检测、得分逻辑和随机球速。
