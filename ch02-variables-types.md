## 2.1 变量：程序中的“命名抽屉”

变量是命名的内存块，用于存储数据。定义变量时必须指定**类型**和**名称**。

### 2.1.1 初始化方式：哪种最安全？

C++ 提供了多种初始化方式，其中**列表初始化 `{}`** 是现代 C++ 推荐的方式，因为它能防止**窄化转换**（例如把小数赋给整数变量）。

```cpp
int a{};        // 值初始化：a = 0
int b{42};      // 列表初始化：b = 42
int c = 42;     // 拷贝初始化（传统）
int d(42);      // 直接初始化（传统）

// 窄化转换被阻止
int e{3.14};    // ❌ 编译错误：double 到 int 会丢失数据
int f = 3.14;   // ⚠️ 合法，但 f 被截断为 3（不推荐）
```

> 完整版第2章会详细对比 5 种初始化方式，并解释为什么列表初始化是首选。

### 2.1.2 `auto`：让编译器帮你写类型

`auto` 关键字让编译器根据初始值自动推导类型，简化代码，尤其适合复杂类型（如迭代器）。

```cpp
auto x = 42;        // x 是 int
auto y = 3.14;      // y 是 double
auto z = x + y;     // z 是 double（表达式类型推导）

#include <vector>
std::vector<int> vec{1,2,3};
auto it = vec.begin();   // it 的类型是 std::vector<int>::iterator（无需手写）
```

> 注意：`auto` 会忽略顶层 `const` 和引用，如果需要保留，可以用 `const auto&` 或 `decltype(auto)`。

### 2.1.3 `decltype`：获取表达式的类型

`decltype` 在编译期推导表达式的类型，并**保留 `const` 和引用**。

```cpp
int a = 5;
decltype(a) b = a;      // b 是 int
decltype((a)) c = a;    // c 是 int&（加了括号变成引用）

const int ci = 10;
decltype(ci) d = ci;    // d 是 const int
auto e = ci;            // e 是 int（丢失 const）
```

**`decltype(auto)`** 结合两者优势，完美保留类型细节，常用于函数返回值。

## 2.2 基本数据类型：内存里的“容器”

C++ 提供丰富的基本类型，每种类型决定了数据的存储大小和取值范围。

| 类型        | 典型大小 | 取值范围（约）                     |
| ----------- | -------- | ---------------------------------- |
| `bool`      | 1 字节   | `true` / `false`                   |
| `char`      | 1 字节   | -128 ~ 127 或 0 ~ 255（由实现决定） |
| `int`       | 4 字节   | -2.1×10⁹ ~ 2.1×10⁹                |
| `unsigned`  | 4 字节   | 0 ~ 4.2×10⁹                       |
| `double`    | 8 字节   | 约 15–16 位有效数字                |

使用 `sizeof` 可以查看任何类型或变量的实际字节数：

```cpp
#include <print>
int main() {
    std::println("int 占 {} 字节", sizeof(int));
    std::println("double 占 {} 字节", sizeof(double));
}
```

> 完整版包含所有整数类型（`short`、`long`、`long long`）、字符类型（`wchar_t`、`char16_t`、`char32_t`）、浮点精度陷阱、IEEE 754 存储原理。

## 2.3 类型转换：小心“隐形杀手”

### 2.3.1 隐式转换

混合类型运算时，编译器会自动转换，但可能产生意外结果：

```cpp
unsigned u = 10;
int i = -20;
std::println("{}", u + i);   // 输出 4294967286（不是 -10！）
```

原因：`i` 被转换为 `unsigned`，负数变成很大的正数。

### 2.3.2 强制转换（推荐 `static_cast`）

```cpp
double pi = 3.14159;
int intPi = static_cast<int>(pi);   // intPi = 3（截断）
```

> 完整版第2章会系统讲解 4 种 C++ 风格转换（`static_cast`、`dynamic_cast`、`const_cast`、`reinterpret_cast`）及其适用场景。

## 2.4 枚举类型：让代码更语义化

### 2.4.1 强类型枚举 `enum class`（推荐）

```cpp
enum class Color { Red, Green, Blue };
Color c = Color::Red;          // 必须带作用域
int n = static_cast<int>(c);   // 显式转换，不会隐式转成整数

// 错误示例：
// Color c2 = Red;     // ❌ 编译错误：Red 不在作用域内
// if (c == 0) {}      // ❌ 编译错误：不能与整数比较
```

**优点**：类型安全、防止命名污染、可指定底层类型。

### 2.4.2 传统 `enum`（不推荐）

```cpp
enum Fruit { Apple, Banana };   // Apple 和 Banana 泄漏到外层作用域
int x = Apple;                  // 隐式转换，容易混乱
```

## 2.5 作用域与生命周期：变量在哪里“活着”

- **全局变量**：定义在函数外，整个程序都可访问，程序结束时才销毁。未初始化默认为 `0`。
- **局部变量**：定义在函数或代码块 `{}` 内，只在块内可见，离开块即销毁。未初始化值不确定（危险！）。
- **静态局部变量**：用 `static` 修饰，生命周期贯穿程序运行，但作用域仍限于块内。

```cpp
#include <print>

int global = 100;          // 全局

void test() {
    static int counter = 0; // 静态局部，只初始化一次
    counter++;
    std::println("counter = {}", counter);
}

int main() {
    int local = 200;        // 局部
    std::println("global = {}, local = {}", ::global, local);
    test();  // counter = 1
    test();  // counter = 2
}
```
